def ensure_dataset(dataset='rohan0301/unsupervised-learning-on-country-data', dest_dir=DATA_DIR):
    # Procura arquivo localmente (case-insensitive)
    candidates = list(dest_dir.glob('Country-data.csv')) + list(dest_dir.glob('country-data.csv'))
    if candidates:
        print('Encontrado arquivo local:', candidates[0])
        return candidates[0]

    kaggle_json = Path.home() / '.kaggle' / 'kaggle.json'
    kaggle_cli = shutil.which('kaggle')

    if kaggle_cli and kaggle_json.exists():
        print('Tentando baixar via Kaggle CLI...')

            # Baixa e descompacta no diretrio data/

Parte 3: Clusterizao - K-Mdias (k=3)

  • Ajustar KMeans em dados padronizados.
  • Calcular centrides e medides (pas mais prximo ao centride) por cluster.
  • Visualizar no espao PCA (2D).

Dados usados e variveis-chave:

  • df_scaled_features: dados padronizados gerados com StandardScaler a partir de num_cols.
  • labels_kmeans: rtulos dos clusters atribudos pelo KMeans.
  • df_kmeans['cluster_kmeans']: coluna com o cluster de cada pas (para agregaes e interpretao).
  • perfil_clusters: mdias das variveis por cluster (perfil socioeconmico/sade tpico).

Medide por cluster (interpretao):

  • Funo calcula o pas mais prximo do centride no espao padronizado (menor distncia euclidiana).
  • til para comunicao: um pas representativo por cluster ajuda a explicar o perfil do grupo.

Consideraes do K-Means:

  • Assume agrupamentos aproximadamente esfricos/convexos e usa distncia Euclidiana.
  • Sensvel a outliers e escala: por isso padronizamos previamente.
  • n_init=10: mltiplas inicializaes para reduzir risco de mnimo local.

# Funo para medides: pas mais prximo ao centride do cluster (no espao padronizado)
def compute_medoids_from_kmeans(X_scaled_df, labels, centroids, countries):
    medoid_indices = {}
    medoid_countries = {}
    X = X_scaled_df.values
    for c in np.unique(labels):
        idx = np.where(labels == c)[0]
        Xc = X[idx]
        centroid = centroids[c]
        dists = np.linalg.norm(Xc - centroid, axis=1)
        local_min = np.argmin(dists)
        medoid_idx = idx[local_min]
        medoid_indices[c] = medoid_idx
        medoid_countries[c] = countries[medoid_idx]
    return medoid_indices, medoid_countries

countries = df[col_country].values
medoid_idx_map, medoid_country_map = compute_medoids_from_kmeans(
    df_scaled_features, labels_kmeans, kmeans.cluster_centers_, countries
)

print('Medides por cluster (KMeans):')
for c in sorted(medoid_country_map.keys()):
    print(f'Cluster {c}:', medoid_country_map[c])

Clusterizao Hierrquica + Dendrograma

  • Mtodo: linkage='ward' minimiza a varincia intracluster e usa distncia euclidiana. Por isso, essencial padronizar as variveis (StandardScaler) antes de aplicar.
  • Guia de leitura do dendrograma:
    • Altura representa a distncia de fuso: junes altas indicam grupos mais distintos; junes baixas indicam subgrupos prximos.
    • O truncamento (p=5) mostra apenas os nveis superiores; se precisar ver todos os nveis e microgrupos, remova o truncamento.
    • A escolha de k pode ser feita por um corte horizontal onde haja um "gap" claro nas alturas. O nmero de ramos interceptados = quantidade de clusters.
  • AgglomerativeClustering com k=3 usado para comparar com KMeans e verificar a estabilidade dos grupos sob outro critrio.
  • Avaliao: utilize o ndice de silhouette e a tabela de contingncia (crosstab) KMeans vs Hierrquico para medir concordncia e qualidade.
    • Silhouette ~0.250.35 indica separao moderada; valores maiores sugerem separao mais ntida.
    • No crosstab, valores altos na diagonal indicam forte concordncia entre mtodos; valores fora da diagonal indicam reagrupamentos.
  • Diferenas esperadas entre KMeans e Hierrquico: sensibilidade a outliers, formas no esfricas dos clusters, e efeitos de inicializao (KMeans). Ward tende a favorecer grupos compactos.

Parte 4: Escolha de Algoritmos

Etapas do algoritmo K-mdias (at a convergncia)

  1. Inicializao: escolher "k" centrides (aleatoriamente ou via heursticas).
  2. Atribuio: associar cada ponto ao centride mais prximo (por distncia, e.g., Euclidiana).
  3. Atualizao: recalcular cada centride como a mdia dos pontos atribudos ao cluster.
  4. Convergncia: repetir etapas 23 at que os centrides mudem muito pouco (ou atingir max de iteraes).

A seguir, implementamos uma variante que "representa" cada cluster pelo medide (ponto real mais prximo ao seu baricentro) em todas as iteraes.

In[13]:
from sklearn.metrics import pairwise_distances

def k_medoids(X, k, max_iter=200, random_state=42):
    '''
    Implementao simplificada de K-medoids:
    - Inicializa medides (pontos reais) aleatrios.
    - Atribui cada ponto ao medide mais prximo.
    - Atualiza cada medide escolhendo o ponto do cluster que minimiza a soma de distncias aos demais.
    - Repete at estabilizar ou atingir max_iter.
    '''
    rng = np.random.default_rng(random_state)
    n = X.shape[0]
    # Inicializao por amostragem sem reposio
    medoid_indices = rng.choice(n, size=k, replace=False)
    D = pairwise_distances(X, X)  # matriz de distncias completa

    for it in range(max_iter):
        # Atribuio
        distances_to_medoids = D[:, medoid_indices]  # (n, k)
        labels = np.argmin(distances_to_medoids, axis=1)

        updated = False
        # Atualizao por cluster
        for c in range(k):
            cluster_points = np.where(labels == c)[0]
            if len(cluster_points) == 0:
                continue
            # Soma das distncias de cada candidato aos outros pontos do cluster
            intra_D = D[np.ix_(cluster_points, cluster_points)]
            costs = intra_D.sum(axis=1)
            best_idx_in_cluster = cluster_points[np.argmin(costs)]
            if best_idx_in_cluster != medoid_indices[c]:
                medoid_indices[c] = best_idx_in_cluster
                updated = True
        if not updated:
            break
    return labels, medoid_indices

K-mdias sensvel a outliers. Explique.

  • O K-mdias utiliza a mdia dos pontos para definir os centrides. A mdia fortemente afetada por valores extremos; portanto, poucos outliers podem deslocar significativamente o centride de um cluster, levando a atribuies incorretas e deteriorando a qualidade dos agrupamentos.

Por que DBSCAN mais robusto a outliers?

  • O DBSCAN define clusters como regies de alta densidade separadas por regies de baixa densidade. Pontos isolados (outliers) tendem a ser classificados como "rudo" (label -1) e no influenciam o centro do cluster. Alm disso, no pressupe formato globular nem exige nmero de clusters pr-definido, sendo mais robusto a diferentes formas e presena de outliers.
